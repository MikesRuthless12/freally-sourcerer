Install & First Run

Freally runs as two pieces: a background daemon (freally-indexd) that keeps the index up to date, and the desktop app that connects to it. Installation sets both up automatically.

  1. Download the installer for your platform from the Releases page: .exe for Windows, .dmg for macOS, or .AppImage / .deb for Linux.
  2. Run the installer. On Windows, run Freally_0.19.84_x64-setup.exe and follow the prompts. On macOS, open the .dmg and drag Freally to Applications. On Linux, make the AppImage executable (chmod +x Freally*.AppImage) and run it, or install the .deb with sudo dpkg -i.
  3. Launch Freally. The first-run wizard opens automatically.
  4. Pick your index roots. In the wizard, select the folders you want Freally to index — for example, your home folder or a specific project drive. You can add more roots later in Settings.
  5. Set a global hotkey. Choose the keyboard shortcut that will summon Freally from anywhere (default is Alt+Space on Windows/Linux). You can change this at any time in Settings.
  6. Choose a locale and theme. Pick your preferred language and Light / Dark / System theme, then click Finish.
  7. Wait for the initial index. Freally crawls your chosen folders in the background. A progress badge in the status bar shows how many files have been indexed. Typical NVMe drives index at 50,000+ files per second — a 1 M-file disk is fully indexed in under a minute. You can search during this time; results populate against whatever has been indexed so far.
Tip: The freally-indexd daemon registers as a Windows Service, macOS launchd agent, or systemd user unit so it starts automatically at login — you never need to launch the daemon manually.

Performing a Search

The search bar is the entire interface. Type anything — a filename fragment, a word from inside a document, an audio attribute — and results appear within milliseconds.

  1. Open Freally. Click the app in your dock/taskbar, or press your global hotkey from anywhere.
  2. Start typing in the search bar. Results stream in as you type, grouped by lens: FILES, CONTENT, AUDIO, and YOU MIGHT MEAN. Each group shows how long it took (e.g., 14 ms).
  3. Narrow your results by typing more characters. Each additional character filters the existing results instantly — no need to press Enter.
  4. Click any result row to select it, or use the arrow keys to move through results.
  5. Open a file by double-clicking its row, or pressing Enter on a selected row. Freally uses the OS default handler for the file type.
  6. Reveal a file in Finder / Explorer / your file manager by pressing Ctrl+R (Windows/Linux) or ⌘R (macOS) with a row selected.
  7. Clear the search bar with Escape to return to the empty/bookmarks view.
Tip: An "Index ready" badge in the status bar confirms the initial crawl is complete. A "Still indexing" badge means results are live but partial — all newly indexed files will appear without a restart.

Search Lenses

Freally fans out across four independent lenses in parallel. Every search runs all four simultaneously unless you narrow to one with a prefix.

  1. Filename lens — finds files by name, path, glob, or regex. This is the fastest lens (P99 under 16 ms on a 5-million-file disk). Results appear under the FILES heading.
  2. Content lens — searches the full text inside files: PDFs, Office documents, Markdown, source code in 32 programming languages, archives (contents peeked without extraction), and structured files like JSON, CSV, and YAML. Results appear under CONTENT with matching snippets.
  3. Audio lens — searches audio files by their acoustic attributes: duration, codec, sample rate, channels, integrated LUFS loudness, true peak, silence ratio, and dynamic range. Results appear under AUDIO. Use query modifiers like lufs:<-14 or codec:flac to target specific attributes.
  4. Similarity lens — finds files whose names are similar to your query using bigram MinHash. Useful when you know a file exists but can't remember the exact name. Results appear under YOU MIGHT MEAN with a similarity score (e.g., similarity 0.91).
  5. Target a specific lens with a prefix: type name:, content:, audio:, or similar: before your query to restrict results to that lens only.
Tip: You can toggle which lenses appear by default in Settings → General → Home → Default lens visibility.

Query Syntax

Freally supports everything you know from voidtools' Everything, plus additional modifiers for the content, audio, and similarity lenses.

Basic patterns

  1. Literal substring: type any word or phrase — e.g., budget report — to find files whose name or content contains those words.
  2. Wildcards: use * for any sequence of characters and ? for a single character — e.g., invoice_202*.pdf.
  3. Regex: prefix with / or enable regex mode with the toolbar toggle — e.g., /invoice_\d{4}\.pdf.
  4. Boolean: combine terms with AND, OR, NOT (or ! prefix) — e.g., report AND NOT draft.
  5. Grouping: use parentheses — e.g., (budget OR forecast) AND 2026.

  2. Enable Regex mode with the /…/ toggle to write full regular expressions.
Example — find loud FLAC masters longer than 3 minutes:

Quick Filters

Quick filters are one-click presets that restrict results to a file category without writing a query. They sit in a palette below the search bar.

  1. Click a filter chip to activate it: Audio, Video, Image, Document, Executable, or Archive.
  2. Combine a filter with a query. Click Audio and then type lufs:<-16 — the filter restricts to audio files and the modifier further narrows by loudness.
  3. Deactivate a filter by clicking it again, or by pressing Escape to clear both the filter and the search bar.
  4. Create a custom filter by going to Settings → General → Results and clicking Add filter. Give it a name and a query expression — for example, a filter called "Big Videos" with the expression ext:mp4 OR ext:mkv size:>1gb. Custom filters appear alongside the built-in chips.

Bookmarks & Saved Searches

Bookmarks let you save any search query for instant recall — equivalent to voidtools' Everything saved searches.

  1. Save the current query as a bookmark: with a query in the search bar, click the Bookmarks dropdown (the star icon near the search bar) and choose Save current search….
  2. Name your bookmark in the dialog that appears, then click Save.
  3. Recall a bookmark by clicking the Bookmarks dropdown and selecting it from the list — the saved query populates the search bar and results load immediately.
  4. Organize bookmarks by selecting Organize… at the bottom of the dropdown. Drag to reorder, click the pencil icon to rename, or click the trash icon to delete.
  5. Use a bookmark at startup: go to Settings → General → Home → Search and paste your saved query into the Custom field so it runs automatically when Freally opens.
Tip: Bookmarks appear on the empty-state screen (when the search bar is clear) so your most-used searches are always one click away.

Working with Results

Every result row is a live file on your disk. Freally gives you direct access to it without opening a file manager.

  1. Open a file: double-click a row, or press Enter with it selected. Freally hands it off to the OS default handler.
  2. Reveal in Finder / Explorer: select a row and press Ctrl+R (Windows/Linux) or ⌘R (macOS), or right-click and choose Reveal.
  3. Copy the file path: right-click a row and choose Copy path, or press Ctrl+Shift+C. Copy name copies just the filename without the path.
  4. Drag a file: click and drag any result row into another app — a text editor, terminal, email client, and so on. Freally uses the OS-native drag protocol so the file drops exactly as if you dragged it from Finder or Explorer.
  5. Delete a file: right-click and choose Delete. Freally asks for confirmation, then sends the file to the OS Trash / Recycle Bin.
  6. Sort results: click any column header (Name, Path, Size, Modified, Type, Extension) to sort ascending; click again to sort descending.
  7. Resize columns: drag the divider between column headers to resize them. Column widths are saved per view profile.
  8. Switch views: use the toolbar view buttons to switch between Compact, Comfortable, Details, and Thumbnails.

Preview Pane

The preview pane shows a live preview of the selected file without opening a separate application.

  1. Open the preview pane by pressing Ctrl+P (Windows/Linux) or ⌘P (macOS), or by clicking View → Preview Pane in the menu.
  2. Select a file row — the preview updates immediately to show the file's content. Images render as a thumbnail, PDFs and documents show a text preview, audio files show waveform metadata, and code files show syntax-highlighted text.
  3. Resize the pane by dragging the divider between the results list and the preview area.
  4. Close the preview pane by pressing Ctrl+P / ⌘P again, or by dragging the divider all the way to the right.
Tip: Enable Thumbnails view to see image thumbnails directly in the result rows without opening the preview pane.

Global Hotkey

The global hotkey summons Freally from any application — no need to find it in the taskbar or dock.

  1. Default hotkeys: Alt+Space on Windows and Linux; the hotkey you chose during the first-run wizard on macOS.
  2. Change the hotkey: go to Settings → General → Keyboard. Click the current shortcut display, press your desired key combination, and click Apply.
  3. Using the hotkey: press the hotkey from any app. Freally appears in the foreground with the search bar focused. Press Escape to hide it and return to your previous app.
  4. If the hotkey conflicts with another app, open Settings and choose a different combination. Combinations involving Ctrl, Alt, Shift, and a letter or function key are least likely to conflict.
Tip: Freally also responds to the freally://search?q=your+query URL protocol — paste it into a browser or terminal to open Freally pre-loaded with a query.

Settings

Settings is a full-featured dialog — modeled on voidtools' Everything Options — with a left-tree navigation and 26 panels. Open it with the ⚙ icon (top-right of the main window) or via Edit → Preferences.

Navigating Settings

  1. Open Settings with the ⚙ button or the keyboard shortcut shown in the menu.
  2. Navigate the tree on the left to reach any panel. The tree is grouped into: General, History, Indexes, Lenses, Network, Privacy & Updates, Logs & Debug, Backup/Export/Reset, Locale, and About.
  3. Filter panels by typing in the search box above the tree — it filters panel names and individual option labels.
  4. Apply changes without closing by clicking Apply. Click OK to apply and close. Click Cancel to discard all unsaved changes. Use Restore Defaults (bottom-left) to reset the current panel only.

Key panels

  1. General → UI: change the app theme (Light / Dark / System), set result row density (Compact or Comfortable), and control tray/menu-bar behavior.
  2. General → Search: configure DSL behavior — boolean operator precedence, regex auto-detect, wildcard matching rules, and Strict Everything syntax mode for maximum compatibility with Everything-style queries.
  3. General → Keyboard: change the global hotkey.
  4. Indexes → Volumes: see all detected volumes (NTFS, APFS, ext4, Btrfs, etc.) and enable or disable indexing per volume.
  5. Indexes → Folders: add or remove index roots — the top-level folders Freally crawls.
  6. Indexes → Exclude: exclude specific paths, extensions, or filename patterns from the index (e.g., node_modules, *.tmp).
  7. Lenses → Content: configure which file types the content extractor processes and set a maximum file size for extraction.
  8. Lenses → Audio: enable or disable EBU R128 LUFS analysis and set the silence-threshold level used when calculating silence ratios.
  9. Network → HTTP / HTTPS Server: enable the optional API server, set the port and authentication token, and configure the CORS allowlist for browser-based search.
  10. Privacy & Updates: review the zero-telemetry guarantee and opt out of (or back into) auto-update checking.
  11. Locale: change the UI language. All 18 included languages switch live — no restart needed. Arabic automatically flips the layout to right-to-left.

Language & Locale

Freally ships in 18 languages. The entire UI — menus, settings, status bar, bookmarks, previews — switches instantly when you change the locale.

  1. Open Settings → Locale.
  2. Choose your language from the dropdown. Languages are listed in their native names (e.g., Español, Deutsch, 日本語).
  3. Click Apply. The UI redraws in the new language immediately — no restart required.
  4. Arabic (العربية) automatically flips the entire layout to right-to-left when selected.
Available languages: English, Spanish, German, French, Italian, Dutch, Polish, Portuguese (Brazil), Turkish, Vietnamese, Indonesian, Russian, Ukrainian, Arabic, Hindi, Japanese, Korean, Chinese (Simplified).

Command-Line Interface

The freally CLI is a thin client over the same query engine as the desktop app — equivalent to Everything's es.exe. It's useful for scripting, piping results, or quick terminal searches.

  1. Run a search:
    freally "budget report"
    Prints matching file paths to stdout, one per line.
  2. Use query modifiers:
    freally "ext:flac lufs:<-14 length:>3:00"
  3. Restrict to a lens:
    freally "content:net profit"
  4. Pipe results to other tools:
    freally "ext:log date:>2026-06-01" | xargs grep "ERROR"
  5. Open the URL protocol from any app: navigate to freally://search?q=your+query to open the desktop app focused on that query.
Tip: The CLI connects to the running freally-indexd daemon over the same IPC socket as the desktop app. If the daemon isn't running, start it from Settings → General → UI by enabling Run in background.

Tips & FAQ

Why aren't some files showing up?

  1. Check the status bar at the bottom of the window. If it shows "Indexing…", the initial crawl is still running — wait for it to complete.
  2. Open Settings → Indexes → Folders and confirm the folder containing those files is listed as an index root.
  3. Open Settings → Indexes → Exclude and verify the files aren't matched by an exclusion rule.
  4. If the files are on a separate volume, open Settings → Indexes → Volumes and make sure that volume is enabled for indexing.

How do I search inside archives?

  1. The Content lens automatically peeks inside .zip, .tar, .gz, and other archive formats without extracting them.
  2. Type your search term normally — if the content appears inside an archive, it will show up under CONTENT with the archive file as the result.

How do I reduce index size?

  1. Open Settings → Indexes → Exclude and add high-volume, low-value paths like node_modules, .git, build output folders, or temp directories.
  2. In Settings → Lenses → Content, reduce the maximum file size for content extraction, or disable extraction for file types you don't need to search inside.

How do I search a remote machine?

  1. On the remote machine, open Settings → Network → HTTP / HTTPS Server, enable the server, set a port and an authentication token, and click Apply.
  2. On your local machine, open Settings → General → Home → Index and select HTTPS API endpoint. Enter the remote machine's URL and token.
  3. Queries now run against the remote index. Results and file operations are routed back over the API.

How do I keep my data private?

  1. Freally makes zero outbound network calls by default. Your index never leaves your machine.
  2. The HTTPS API server is opt-in and disabled by default — enable it only if you specifically need remote access.
  3. Auto-update is the single, opt-out exception. You can disable it in Settings → Privacy & Updates.

How do I reset the index?

  1. Open Settings → Backup, Export, Reset.
  2. Click Reset index and confirm. The daemon stops, clears the on-disk index, and immediately begins a fresh crawl.
Keyboard shortcut reference: hover over any button or toolbar icon to see its keyboard shortcut. A full list is also available in Settings → General → Keyboard.
